VCLValueOf

This class represents the value-of {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This object contains a XPath expression or it references a defined variable ({@link de.xplib.nexd.api.vcl.VCLVariable}).

  • XPath 
    <vcl:value-of select="/foo[@bar='manuel']/*" />
  • Variable 
    <vcl:value-of select="$foobar" />

This class implements a part of the GoF Composite pattern (163).
VCLSchemaCompileVisitor

This represents a visitor (GOF Visitor Pattern [331]) that creates pre compiled resources from a given {@link de.xplib.nexd.api.vcl.VCLSchema}.

VCLAttr

This class represents the dynamic {@link org.w3c.dom.Attr} instances everywhere in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This attribute contains a XPath expression or it references a defined variable ({@link de.xplib.nexd.api.vcl.VCLVariable}).

  • XPath 
    <foo vcl:bar="/foo[@bar='manuel']/*" />

will become

<foo bar="[Any value here]" />
  • Variable 
    <foo vcl:bar="$foobar" />

will become

<foo bar="[Any value here]" />

This interface implements a part of the GoF Composite pattern (163).

VCLPreCompiler

This is the default pre compiler for a Virtual Collection Language Schema. It creates pre compiled resources from the given {@link VCLSchema}, the so called {@link _de.xplib.nexd.api.pcvr.PCVResource} instances. These pre compiled resources contain both the structure of the {@link de.xplib.nexd.api.vcl.VCLSchema} and the content of the resulting {@link _de.xplib.nexd.api.VirtualResource}.
UndeclaredVariableException

This Exception is thrown if a variable is accessed that was not defined before.
VCLVariable

This class represents the variable {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This object contains a XPath expression and a name which is used to access the content later.

<vcl:variable name="foo" select="/foo[@bar='manuel']/*" />

Scope

The following example shows the scope of a variable in a Virtual Collection Language Schema

<vcl:collection match="/db/foo">                                  (1)
<vcl:variable name="var1" select="//*[@id=1]" />
...
<vcl:collection match="/db/bar">                                (2)
<vcl:variable name="var11" select="//*[@type='int']" />
...
<vcl:collection match="/db/foobar">                           (3)
<vcl:variable name="var111" select="//*[@name='manu']" />
...
</vcl:collection>
</vcl:collection>
<vcl:collection match="/db/foobar">                             (4)
<vcl:variable name="var12" select="//*[@name='manuel']" />
...
</vcl:collection>
</vcl:collection>
  • var1 is visible everywhere, this means in the collections 1, 2, 3 and 4.
  • var11 is visible in the collections 2 and 3.
  • var111 is only visible in 3.
  • var12 is only visible in 4.

VCLSchema

This class represents the schema {@link org.w3c.dom.Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This is the first {@link org.w3c.dom.Element} in each Virtual Collection Language Schema.

This class implements a part of the GoF Composite pattern (163).
VCLNode

This interface is the base for all types of a {@link de.xplib.nexd.api.vcl.VCLSchema} tree.

This interface implements a part of the GoF Composite pattern (163) and the Visitor pattern (331). The visitor is used for multiple tasks, it can perform syntax checks, transaction locks, compiling the schema and may more.

AbstractVCLParser

This is the abstract base class for parser that parses a Virtual Collection Language Schema.

The interface of this class is inspired by {@link javax.xml.parsers.DocumentBuilder} class.
VCLCollectionReference

This class represents the collection {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This object references a {@link org.xmldb.api.base.Collection} in the xml database

  • match

    This Element selects the {@link org.xmldb.api.base.Collection} bar in its parent {@link org.xmldb.api.base.Collection} foo. 

    <vcl:collection match="/foo/bar">
</vcl:collection>
  • match + select

    This Element selects the {@link org.xmldb.api.base.Collection} bar in its parent {@link org.xmldb.api.base.Collection} foo. But it just returns that {@link org.xmldb.api.modules.XMLResource} objects where the XPath expression in the @select matches. 

    <vcl:collection match="/foo/bar" select="/foo[@bar='manuel']">
</vcl:collection>

This class implements a part of the GoF Composite pattern (163).
