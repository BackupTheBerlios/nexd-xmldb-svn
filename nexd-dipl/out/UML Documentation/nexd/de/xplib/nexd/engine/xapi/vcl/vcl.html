Class Summary
VCLAttrImpl

This class represents the dynamic {@link org.w3c.dom.Attr} instances everywhere in the DOM {@link org.w3c.dom.Document} of the parsed {@link de.xplib.nexd.api.vcl.VCLSchema}. This attribute contains a XPath expression or it references a defined variable ({@link de.xplib.nexd.api.vcl.VCLVariable}).

  • XPath 
    <foo vcl:bar="/foo[@bar='manuel']/*" />

will become

<foo bar="[Any value here]" />
  • Variable 
    <foo vcl:bar="$foobar" />

will become

<foo bar="[Any value here]" />

This interface implements a part of the GoF Composite pattern (163).

VCLSchemaImpl

This class represents the schema {@link org.w3c.dom.Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link VCLSchema}. This is the first {@link org.w3c.dom.Element} in each Virtual Collection Language Schema.

This class implements a part of the GoF Composite pattern (163).
VCLValueOfImpl

This class represents the value-of {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link de.xplib.nexd.api.vcl.VCLSchema}. This object contains a XPath expression or it references a defined variable ({@link de.xplib.nexd.api.vcl.VCLVariable}).

  • XPath 
    <vcl:value-of select="/foo[@bar='manuel']/*" />
  • Variable

VCLCollectionReferenceImpl

This class represents the collection {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link de.xplib.nexd.api.vcl.VCLSchema}. This object references a {@link org.xmldb.api.base.Collection} in the xml database.

  • match

    This Element selects the {@link org.xmldb.api.base.Collection} bar in its parent {@link org.xmldb.api.base.Collection} foo. 

    <vcl:collection match="/foo/bar">
</vcl:collection>
  • match + select

    This Element selects the {@link org.xmldb.api.base.Collection} bar in its parent {@link org.xmldb.api.base.Collection} foo. But it just returns that {@link org.xmldb.api.modules.XMLResource} objects where the XPath expression in the @select matches. 

    <vcl:collection match="/foo/bar" select="/foo[@bar='manuel']">
</vcl:collection>

This class implements a part of the GoF Composite pattern (163).

VCLVariableImpl

This class represents the variable {@link Element} in the DOM {@link org.w3c.dom.Document} of the parsed {@link de.xplib.nexd.api.vcl.VCLSchema}. This object contains a XPath expression and a name which is used to access the content later.

<vcl:variable name="foo" select="/foo[@bar='manuel']/*" />

Scope

The following example shows the scope of a variable in a Virtual Collection Language Schema

<vcl:collection match="/db/foo">                                  (1)
<vcl:variable name="var1" select="//*[@id=1]" />
...
<vcl:collection match="/db/bar">                                (2)
<vcl:variable name="var11" select="//*[@type='int']" />
...
<vcl:collection match="/db/foobar">                           (3)
<vcl:variable name="var111" select="//*[@name='manu']" />
...
</vcl:collection>
</vcl:collection>
<vcl:collection match="/db/foobar">                             (4)
<vcl:variable name="var12" select="//*[@name='manuel']" />
...
</vcl:collection>
</vcl:collection>
  • var1 is visible everywhere, this means in the collections 1, 2, 3 and 4.
  • var11 is visible in the collections 2 and 3.
  • var111 is only visible in 3.
  • var12 is only visible in 4.
.
